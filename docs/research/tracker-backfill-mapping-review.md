- 입력 코퍼스: `historical-candidates-v1.jsonl` 212건
- 생산 매핑: `backfill-v1.json` 146개 청구
- 고유 사용 후보: 140건

- 노드 세트: `nodes-v1.0`

- 출처 코드: `NASA`, `ESA`, `FAA`, `UNOOSA`, `GOVINFO`, `LSA`

## 승인 범위와 검토 책임

사용자는 `nodes-v1.0`과 참조형 백필 진행을 승인했고, WP2.2-A에서 P1 누락 근거와 휴면 전이를 보강했다. 이 매핑 패스에서는 각 행의 사실 메타데이터 검토와 루브릭 경계 검토를 분리해 수행했고, 두 상태가 모두 `APPROVED`인 행만 생산 JSON에 포함했다. 별도의 제3자 검토자가 참여했다고 주장하지 않는다.

## 저작권·저장 정책

생산 매핑에는 제3자 원문, 인용문, 발췌문, HTML, PDF, 이미지, 첨부파일 또는 출처 제목을 저장하지 않는다. 후보 식별자, 공식 URL 참조, 접근일, SHA-256 지문, 발행 경로, 직접 작성한 사건 제목·경계 설명만 사용한다. 원문 응답 바이트는 저장하지 않으며 가져오기 중 네트워크나 LLM 호출도 하지 않는다.

## 판정 규칙

1. 후보의 날짜와 정밀도는 그대로 복사하며 정밀도를 올리지 않는다.
2. 하나의 근거 참조는 해당 `factSummary`가 직접 지지하는 노드에만 사용한다.
3. `SETBACK`, `PROGRAM_CANCELLATION`, `ANNOUNCEMENT_ONLY`, `RETROSPECTIVE`는 상태를 올리지 않는다.
4. 부분 재사용, 지상 전용 계통, 단일 부품, 계획·계약만으로 운영 등급을 추론하지 않는다.
5. 통합 노드는 동일 캠페인과 동일 운영 구간의 직접 근거가 없으면 0을 유지한다.
6. 여섯 출처는 모두 Tier 1 공공기관이므로 현재 파생 규칙상 `OFFICIAL`이다.

## P1 현재상태 감사

아래 값은 생산 리소스를 fresh H2에 날짜순 재생해 확인했다. `진척 청구`는 비진척 이벤트 타입을 제외한 수이며, 휴면은 입증된 최고 레벨을 없애지 않고 현행 `params-v1` 계수만 적용한다.

## 미선정과 거절 처리

72개 후보는 중복 사건, 경계 밖 사례, 직접성이 약한 계획을 억지로 노드 상태로 만들지 않기 위해 미선정했다. 이 후보들은 `READY_FOR_MAPPING` 상태로 원본 코퍼스에 남아 향후 노드 또는 루브릭 개정 때 다시 검토할 수 있다. 이번 생산 JSON에는 사실 검토 또는 루브릭 검토가 `REJECTED`인 행이 없다.
