> **v2.10 → v2.11 변경 요약 (finish line 재정의 — 도달 가능성 감사 반영):**
> 1. **finish line을 기간 기준 자율성 실증으로 재정의** — "외부 보급 없이 무기한 생존"(관측 불가 술어 + 완전 산업 자급자족으로 오독 시 세기 단위) 폐기 → **"연속 26개월(화성 회합 주기 1회) 무보급 정상 운영 실증"** (1절)
> 2. **"완전 산업 자급자족"을 테라포밍과 같은 세기 단위 주제로 명시 스코프 아웃** — 요구되는 것은 질량 대분율 루프(물·산소·식량·부품)의 현지 폐쇄이며, 저질량 고복잡도 재화(반도체·의약품)의 비축·정기 보급은 허용 (1절)
> 3. **엔트로피 불가능성 비판 기각 기록** — 정착지는 닫힌 계가 아니라 열린 계(에너지 유입 + 행성 물질). 지구 생물권이 실존 증명. 단, 소규모 인구의 완전 자급자족 비현실성 지적은 수용되어 1·2번으로 해소 (1절)
> 4. **사후 이정표 승계 확장** — 52개월(보급 창 1회 유실 내성) → 자급률 상승 → 완전 산업 자급자족 (1절)
> 5. **(G0 소급 반영)** 사건 유형에 `제도·시장 전진(INSTITUTIONAL_ADVANCE)` 추가(12종) — 구현 설계에서 필라 6의 *전진* 사건(조약 채택, 시범 계약, 최초 매출)에 맞는 유형 부재가 발견되어 게이트 승인으로 추가 (6절)

> **v2.9 → v2.10 변경 요약 (인식론 감사 — 정직성 표기):**
> 1. **ETA의 인식론적 지위 명시** — ETA는 예보(forecast)가 아니라 "명시된 가정 하의 시나리오 투영(projection)". 구간은 **"모델 내 80%"**(모형 불확실성 제외)로 라벨링 (8.4, 11절)
> 2. **$/kg의 실체 명시** — 관측 가능한 것은 '비용'이 아니라 '공표 가격'. 가격 기반 추정치임을 명시하고, 수송 경제성 목표 임계값은 선언된 가정임을 게시 (3.2, 8.3절)
> 3. **측정 vs 구성 지수의 구분** — 문명 단위 TRL은 max-over-actors 정의 관습(HDI류 구성 지수)임을 명시 + 집계 한계 각주 (3.3절), 방법론 페이지에 구분 게시 요건 추가 (9절)
> 4. **마이크로 예측 확률 생성 규칙**은 미명세 갭으로 확인 — 구현 설계(시행계획 WP4.5)에서 위험률(hazard rate) 계열로 정의하도록 위임 (9절)
>
> **v2.8 → v2.9 변경 요약 (6차 검토 — 트레이드오프 판정 및 종결):**
> 1. **명칭 최종 교정: "1측정 + 2앵커" → "1측정 + 1앵커 + 1관측"** — 자동 효과가 없는 Layer A를 '앵커'로 부르는 것은 과장. 관측 게이지로 완전 분리 (2, 3절)
> 2. **서킷 브레이커 신설** — 골든셋 일치율 임계 미달 또는 관리도 위반 시 상태 갱신을 **자동 동결**(사건은 잠정 큐에 적체), 인간 검수로 해제. 멱등 상태 모델 덕에 동결 비용은 "지연"일 뿐 오염이 아니다 (9절)
> 3. **TRL/EGL 8~9 전진의 인간 검수 필수 명문화** — 채점 구조상 이미 보장되던 것(base(8)×0.9=8.1≥8 → 검수 큐)을 명시 규칙으로 승격 (7절)
> 4. Tier 1 "관료적 낙관주의" 권고(경성 지표의 사건 게이트화)는 **기각** — 계획 발표 무효화·증거 인용 의무·고임팩트 인간 검수·분기 B쌍 감사의 4겹 방어가 기존재하고, 월~분기 집계인 Layer B로 개별 사건을 게이트하면 케이던스 불일치로 정당한 갱신이 수개월 지연됨

## 1. 비전과 도달 조건의 조작적 정의

**비전:** 인류가 다행성 종족으로 거듭나는 여정을 실시간 추적하는 AI 관제 대시보드. 매일의 과학/기술 뉴스와 객관적 문명 지표를 결합해 진행 상황을 시각화하고 도달 예상 연도(ETA)를 예측한다.

**도달 조건 (finish line, v2.11 재정의):**

> **"지구 외 천체의 상설 유인 거주지가, 지구로부터의 어떠한 물자 보급도 없이 연속 26개월(화성 회합 주기 1회) 이상 정상 운영과 승무원 건강 유지를 실증한 상태"**
> = 6대 필라의 모든 핵심 역량(capability)이 TRL/EGL 8 이상(실운용 검증 완료)에 도달한 상태 — 각 필라의 "통합 실증" 노드(5절)는 이 26개월 실증 사건 없이는 TRL 8+에 도달할 수 없도록 정의하여, 산문 정의와 조작적 정의를 등치로 고정한다.

**재정의의 근거 (v2.11):**

- **왜 기간 기준인가:** 구 정의 "외부 보급 없이 무기한 생존"은 ① "무기한"이 관측 불가능한 술어이고, ② "완전 자급자족"으로 읽힐 경우 반도체·의약품까지의 산업 자급을 함의해 세기 단위로 밀려나는 문제가 있었다. 기간 기준은 이진 관측 가능하다(해당 구간에 보급이 도착했는가 + 정상 운영이었는가).
- **왜 26개월인가:** 화성 발사 창은 회합 주기 약 26개월마다 열리므로, 보급선 사이의 구조적 공백이 물리적으로 26개월이다 — 이 기준은 "다음 보급 창까지 자력으로 건너는 능력"이라는 물리적 의미를 갖는다. 동시에 현 인류 최고 기록(남극 월동 ~9개월, ISS 비축 수개월)을 크게 상회하는 미실증 수준으로, **현실적(수십 년 스케일 도달 가능) + 높은 기대치(현 기록 ~3배) + 관측 가능**의 세 조건을 충족한다. 달 등 다른 천체의 거주지에도 동일 기준을 적용한다(위치 무관 단일 기준).
- **요구하는 것과 허용하는 것:** 이 기준은 질량 대분율 루프(물·산소 재순환+ISRU, 식량 대부분 현지 생산, 예비 부품 비축+적층 제조)의 현지 폐쇄를 강제하되, 저질량·고복잡도 재화(반도체·의약품 등)의 창 단위 비축과 평시 정기 보급은 허용한다 — 정기 보급은 정착지의 정상 상태다. **완전한 산업 자급자족은 테라포밍과 동급의 세기 단위 주제로 명시적으로 스코프 아웃**하며, 도달 후 이정표로 승계한다.
- **물리적 가능성 (엔트로피 비판 기각 기록):** 정착지는 닫힌 계가 아니라 **열린 계**다 — 에너지(태양광·원자력)가 유입되고 행성 규모의 물질에 접근한다. 열역학 제2법칙은 에너지 유입 없는 고립계에만 질서 유지를 금지하며, 지구 생물권(태양 에너지 + 현지 물질로 40억 년 유지)이 실존 증명이다. "무기한 생존"의 장애물은 물리 법칙이 아니라 공학·정비·규모의 문제였고, 본 재정의가 그 부분을 기간 기준으로 해소한다.
- 테라포밍, 항성 간 이주, 카르다쇼프 Type I 도달, **완전 산업 자급자족**은 finish line에서 **제외**한다. (카르다쇼프 지수는 거시 맥락 게이지 — 3.1절)
- 이 정의는 대시보드에 명시적으로 게시한다. "무엇까지 가면 시계가 0이 되는가"가 제품 신뢰성의 근간이다.

**도달 판정과 종료 규칙 (v2.5 신설):**

- **도달은 외삽이 아니라 상태 조건으로 선언한다.** ETA 회귀(8.2절)는 "아직 도달 전"의 접근 속도를 재는 장치일 뿐이며, 도착 자체는 기술 트리의 상태 검사가 선언한다: 모든 핵심 역량 노드가 `공식 확인` 이상의 검증으로 TRL/EGL 8+에 도달하고, 그 상태가 **12개월 연속 유지**될 것 (일시적 충족-이탈의 깜빡임 방지).
- 도달 선언은 시스템 역사상 최대 임팩트 사건이므로 자동 선언하지 않는다 — 플루크 필터와 **인간 검수를 최종 관문**으로 거친 뒤 카운트다운을 0으로 고정하고 1차 임무를 종료한다.
- **사후 운영 (이정표 승계, v2.11 확장):** 종료 시 finish line을 후속 이정표 사다리로 승계한다 — **52개월 무보급(보급 창 1회 유실 내성)** → 거주 인구 임계값 → 자급률(질량 기준 현지 조달 비율) 상승 → 제2 정착지 → 완전 산업 자급자족(세기 단위). 또는 이미 탑재된 K-지수(3.1절)가 Type I을 향한 장기 게이지로 주 지표 자리를 승계한다. 어느 쪽이든 도달 이전의 시계열과 예측 트랙 레코드는 아카이브로 보존한다.

## 2. 핵심 설계 원칙

| # | 원칙 | 의미 |
|---|------|------|
| 1 | **기사가 아니라 사건(Event)을 평가한다** | 여러 매체의 같은 뉴스는 하나의 사건으로 병합. 점수 인플레이션 원천 차단 |
| 2 | **LLM은 분류하고, 코드가 계산한다** | LLM 출력은 전부 enum/인용문. 0~10점·준비도·ETA 등 모든 숫자는 결정론적 코드 산출물 |
| 3 | **점수 누적이 아니라 상태 갱신이다** | 진행도의 원천은 "기술 트리 각 노드의 현재 TRL"이라는 유계 상태 |
| 4 | **병목이 도착 시간을 결정한다** | 다행성 달성은 AND 조건. 필라 간 평균이 아니라 최소값(병목)이 ETA를 지배 |
| 5 | **모든 판단은 증거 인용을 동반한다** | LLM의 모든 분류 결과는 기사 원문 발췌 인용문이 없으면 기각 |
| 6 | **측정은 앵커로 반증한다 (1측정 + 1앵커 + 1관측)** | 뉴스 기반 측정(Layer C)은 발사 통계 등 비-AI 실측 앵커(Layer B)와 매칭 쌍 단위로 대조하고, K-지수(Layer A)는 자동 효과 없는 장기 관측 게이지로 둔다. B쌍 발산은 하이프 또는 오류의 신호 |

## 3. 측정·앵커·관측 체계 — "1측정 + 1앵커 + 1관측" (One Measurement, One Anchor, One Gauge)

*(구명 "3층 측정 체계" → v2.6 "1측정 + 2앵커" → v2.9 최종 교정 — Layer A는 자동 효과가 없으므로 '앵커'가 아니라 '관측 게이지'다.)* 서로 독립적인 데이터 원천을 가진 3개 레이어로 구성하되, 구조의 정체는 **"1개의 측정(Layer C) + 1개의 반증 앵커(Layer B) + 1개의 장기 관측 게이지(Layer A)"**다. 헤드라인 숫자(ETA)는 AI가 해석하는 Layer C에서 도출되고, **Layer B**는 AI가 개입하지 않는 실측 데이터로서 Layer C의 **계통 편향을 반증·보정**하며, **Layer A**는 자동 효과 없이 문명 스케일의 장기 맥락을 관측한다. AI 해석의 계통 편향은 LLM을 여러 겹 쌓아도 잡을 수 없고 비-AI 앵커와 인간 라벨만 잡을 수 있다는 것이 이 구조의 존재 이유이며, "헤드라인 숫자는 AI 해석 레이어에서 나온다"는 사실 자체를 방법론 페이지에 명시한다.

```
Layer C  [측정]       기술 트리 — 역량별 TRL/EGL 상태     일 단위, 뉴스+AI 분류   "개별 역량이 성숙하고 있는가"
Layer B  [앵커]       경성 지표 — 발사 비용·체류 인일 등   월~분기, 구조화 통계    "우주 활동이 실측으로 동행하는가"
Layer A  [장기 관측]  카르다쇼프 K-지수                    연 단위, 에너지 실측    "문명 스케일의 장기 정합"
```

### 3.1 Layer A — 카르다쇼프 K-지수 (거시 게이지·장기 관측)

**정의:** 칼 세이건의 보간 공식을 사용한다.

```
K = (log₁₀ P − 6) / 10        (P = 인류 문명의 총 에너지 사용률, 와트)
```

- Type I = 10¹⁶ W (K=1.0), Type II = 10²⁶ W, Type III = 10³⁶ W
- 현재 인류: P ≈ 2×10¹³ W → **K ≈ 0.73**
- 에너지 성장률이 연 2% 수준일 때 Type I까지는 2~3세기 이상 — 따라서 **K는 finish line이 아니라 거시 맥락 게이지**다. 다행성 전환은 K 0.73→1.0 여정의 초입에서 일어나는 사건이다.

**데이터 소스:** IEA / Energy Institute Statistical Review / Our World in Data 등 공개 에너지 통계 (연 단위 갱신, 분기 추정 병행). **뉴스도 LLM도 개입하지 않는 순수 실측 지표**라는 점이 핵심 가치다.

**에너지 회계 기준과 디커플링 조항 (v2.7):** P를 1차 에너지 총량으로 그대로 쓰면 재생에너지 전환기에 회계 왜곡이 생긴다 — 화석 발전(변환 효율 30~40%)을 재생으로 대체하면 유효 에너지는 그대로인데 1차 에너지 통계가 줄어 보이는 인공 감소 효과다. 따라서 K 산정은 **대체법(substitution method) 또는 유효 에너지 기준을 명시해 사용**한다. 그럼에도 효율화·탈물질화로 총 에너지가 정체하는 진짜 디커플링은 가능하며, 이 경우 A↔필라 4 쌍의 발산은 하이프가 아니라 **문서화된 정당한 발산**이다 — 연례 정합 리포트에 원인 각주와 함께 기록하고 경보로 처리하지 않는다. "우주 진출에 할당된 에너지 비율" 같은 직접 지표는 공개 통계가 존재하지 않아 채택을 보류하며, Layer B의 추진제 소비·궤도 투입 질량이 그 근사 프록시 역할을 한다.

**역할 (v2.6 수정 — 하이프 감지 역할에서 해제):**
1. **거시 게이지:** 대시보드에 "인류 문명 지수 K = 0.73XX"를 소수점 4자리로 표시하고 연간 상승분을 시각화. (소수점 4자리는 연 단위 변화를 체감시키기 위한 연출 — 실질 정보량은 낮지만 제품적으로 유효)
2. **필라 4의 장기 캡:** K의 연간 변화는 ΔK ≈ 0.001 수준(에너지 성장 ~2%/년 기준)으로, Layer C가 급변하는 10년 구간에도 사실상 평행선이며, 그 변동의 지배 요인도 우주가 아니라 거시 에너지 경제(재생에너지 전환, 경기 변동)다. 따라서 **단기 하이프 감지는 케이던스가 맞는 Layer B의 전담 역할**로 두고, K는 필라 4(자원·에너지) 준비도의 **장기(10년+ 윈도우) 정합 캡**으로만 기능한다 — 연 1회 정합 리포트로 점검하며 분기 경보 트리거에서 제외한다 (3.4절).

**채택하지 않은 대안:** 세이건의 정보량 척도(문명 보유 정보 기반), 배로우 척도(미시 제어 기반) 등은 실측 데이터 확보가 어려워 제외. 카르다쇼프(에너지)는 유일하게 **공신력 있는 연간 실측 통계가 존재**하는 문명 척도다.

### 3.2 Layer B — 경성 지표 (Hard Metrics)

뉴스 해석이 아니라 구조화된 통계·API에서 직접 수집하는 객관 KPI. LLM을 거치지 않는다.

| 지표 | 의미 | 예시 소스 |
|------|------|-----------|
| LEO 발사 비용 ($/kg) | 우주 접근성의 대표 지표. 셔틀 ~$54,000/kg → 팰컨9 ~$2,700/kg → 스타십 목표 <$200/kg. **공표 가격 기반 추정치 — 실제 내부 비용은 기업 비밀로 원리적 관측 불가 (v2.10 명시)** | 공개 분석 자료 (CSIS, NASA 등) |
| 연간 궤도 투입 질량 (upmass) | 우주 활동의 물리적 총량 | BryceTech 분기 리포트, GCAT(J. McDowell) |
| 연간 궤도 발사 횟수 / 성공률 | 발사 케이던스와 신뢰성 | Launch Library 2 API |
| 유인 우주 체류 인일(person-days), 동시 체류 최대 인원 | 인간의 우주 상주 능력 | 공개 미션 데이터 |
| 심우주 유인·화물 미션 수, 지구 외 표면 가동 자산 수 | 심우주 진출 실적 | 기관 공개 데이터 |

**역할 (v2.4 스코프 축소):**
- (a) **매칭된 필라에 대한 실측 검증** — 각 지표는 검증 대상 필라에 명시적으로 매핑한다 ($/kg·발사 케이던스 ↔ 필라 1, 유인 체류 인일 ↔ 필라 2, 시장·투자 지표 ↔ 필라 6). 매핑이 불명확한 총량 지표(예: 통신 위성이 지배하는 upmass 총량)는 참고 지표로만 표시한다 — 최근 발사 통계 폭증의 대부분은 LEO 통신 비즈니스이므로, 이를 "다행성 진행도"로 읽는 것은 오독이다.
- (b) **수송 경제성 ETA** 산출(8.3절) — $/kg **공표 가격 곡선**을 Wright's law(누적 발사량 대비 경험 곡선)로 외삽해 필라 1의 목표 경제성 도달 시점을 추정하는 **필라 1 한정** 교차 검증치. 발사 통계에서 문명 전체 ETA를 뽑지 않는다 (그러려면 "어떤 비용 수준이 정착을 가능케 하는가"라는 모델링 가정이 필요해져 독립성이 무너진다). **목표 경제성 임계값 자체도 실측이 아니라 선언된 가정**이므로 방법론 페이지에 가정으로 게시한다 (v2.10).
- (c) 대시보드의 "숫자로 보는 우주 시대" 섹션.

### 3.3 Layer C — 기술 트리 (뉴스 기반 TRL 상태) — 시스템의 코어

6대 필라 × 필라당 4~8개 핵심 역량 = 약 30~40개 노드. 각 노드는 `{현재 TRL/EGL(1~9), 검증 수준, 신뢰도, 활성/휴면 상태, 최종 갱신일}`을 가진다. 뉴스는 이 상태를 갱신하는 증거로만 작동한다. (파이프라인은 6절, 채점 규칙은 7절)

**측정이 아니라 구성 지수다 (v2.10 명시):** "문명 단위 TRL"은 자연에 존재하는 양이 아니라 **"지구상 누군가가 공개 입증한 최고 수준(max-over-actors)"이라는 정의된 관습**이다 — HDI(인간개발지수)류의 구성 지수와 같은 부류로, 정의가 명시되고 일관 적용되는 한 유효한 데이터화다. 시스템의 *입력*(사건 = 관측 가능한 공개 시연)은 측정이고, TRL 사상·가중치 같은 *중간 구성물*은 지수이며, 이 구분을 방법론 페이지에 게시한다(9절). **집계 한계 각주:** max-over-actors는 실제로는 통합 불가능한 역량(서로 다른 국가·기업의 시스템)을 합산할 수 있어 중간 준비도를 과대평가할 수 있다 — 단, finish line 질문("*누군가가* 해내는가")에 대해서는 올바른 정의다.

### 3.4 앵커 대조 규칙 (매칭 쌍 기반, v2.6 개명)

앵커 대조는 전역("모든 레이어가 같은 방향이어야 한다")이 아니라 **같은 대상을 재는 매칭 쌍에만** 적용한다. 레이어마다 재는 대상과 시간 스케일이 달라 **정당한 발산**이 존재하기 때문이다 (예: 통신 위성 시대에 Layer B 발사 통계가 폭증하고 Layer C 정착 역량이 완만한 것은 하이프가 아니라 정상).

| 매칭 쌍 | 정합 질문 |
|---------|-----------|
| Layer B $/kg·발사 케이던스 ↔ Layer C 필라 1(수송) | 수송 준비도 상승이 실측 비용 하락과 동행하는가 |
| Layer B 유인 체류 인일 ↔ Layer C 필라 2(생명유지) | 생명유지 준비도 상승이 실제 체류 실적과 동행하는가 |
| Layer B 시장·투자 지표 ↔ Layer C 필라 6(경제, EGL) | EGL 상승이 실측 시장 데이터와 동행하는가 |
| Layer A K-지수 ↔ Layer C 필라 4(에너지) | 에너지 준비도의 **장기(10년+ 윈도우)** 추세 정합 — **연 1회, 관측·각주 전용, 발산의 자동 효과 없음 (v2.8)** |

- **B쌍 발산 시 (분기 점검):** ① 내부 경보, ② 해당 구간 사건 분류의 표본 감사, ③ 해당 필라 ETA의 불확실성 구간 자동 확대. **발산을 숨기지 않고 방법론 페이지에 공개**한다 — 이 투명성 자체가 신뢰도 자산이다.
- **A쌍 발산 시 (연 1회, v2.8 격하):** 경보·구간 확대 등 자동 효과 없이 **관측·각주 전용**으로 처리한다. 극단적 에너지 효율화(디커플링) 시나리오에서 K와 우주 개척 궤적은 정당하게 영구 분리될 수 있어 발산의 증거 가치가 낮기 때문이다 — K의 거시 게이지 상징성은 유지하되 앵커 비중은 하향하며, 실질적 반증 앵커 역할은 Layer B가 전담한다.
- **비매칭 발산:** 경보가 아니라 해석 각주로 게시한다.

## 4. 도메인 모델

3계층 구조: **기사(Article) → 사건(Event) → 역량(Capability)**

- **기술 트리 (Capability Tree):** 3.3절의 노드 집합. "인류가 얼마나 왔는가"의 유일한 진실 원천.
- **의존성 그래프 (v2.2 신설):** 역량 노드는 독립적이지 않다 — 노드 간 **선행 조건 간선**을 가진 방향성 비순환 그래프(DAG)로 관리한다. 간선은 두 타입:
  - **AND 간선** (모든 선행 필요): 예 — "장기 유인 표면 체류" ← "정밀 대형 화물 착륙" AND "폐쇄 생태계(ECLSS)"
  - **OR 간선** (대체 경로 중 하나면 충분): 예 — "화성 표면 이륙·귀환"의 추진제 확보 ← "ISRU 추진제 생산" OR "사전 배치 지구 공급 체인"
  - 의존성은 **상태 기록을 차단하지 않는다** — 증거가 입증한 TRL은 그대로 기록한다. 의존성이 제약하는 것은 집계·예측에 쓰이는 **유효 준비도**다(8.1절). 후행 노드는 프로토타입 단계까지는 선행보다 앞서갈 수 있으나, 실운용 성숙은 선행 조건에 묶인다는 현실을 수식으로 반영한다.
  - 5절의 "통합 실증" 노드는 이 DAG에서 다수의 선행 간선이 수렴하는 종착(sink) 노드로 자연스럽게 표현된다.
- **사건 (Event):** 중복 제거된 실제 세계의 발생 건. `{유형, 관련 역량, 주체, 발생일, 검증 수준, 증거 인용문, 출처 기사 목록}`. 타임라인과 점수의 단위.
- **기사 (Article):** RSS로 수집된 원문. 사건에 증거로 첨부. 원문은 보존한다(루브릭 개정 시 전체 재평가용).

**역량의 상태 전이 (v2.1 확장):**

```
잠정(provisional) → 활성(active, TRL 확정) → 휴면(dormant) → 재활성
```

- **휴면(dormancy) 규칙:** 문명 수준에서는 입증된 역량도 소실될 수 있다. **역사적 근거 — 아폴로:** 인류는 1969~72년 유인 달 착륙·초대형 발사체(새턴 V)를 TRL 9로 입증했으나, 프로그램 종료 후 약 50년간 해당 역량을 상실했다. 규칙: 역량을 입증한 운영 프로그램이 종료되고 N년(기본 15년) 내 재시연이 없으면 `휴면`으로 전환한다. **감쇠는 기간 의존 곡선이다 (v2.6 — 일률 ×0.6 폐기):** 휴면 진입 시 ×0.85에서 시작해 10년 경과마다 −0.15, 하한 ×0.40 — 지식·인력·제조 공정의 유실은 시간에 비례하며, 50년 휴면한 역량(새턴 V급)의 재현은 신규 개발에 준하는 램프업을 요구하기 때문이다. **복원은 재시연이 입증한 수준까지만 (v2.6):** 원 TRL의 재시연이면 완전 복원, 하위 단계의 재시연이면 그 단계까지. 복원이 증거를 건너뛰는 일은 없다 — 재현 프로그램의 램프업 과정 자체가 재시연 이전 구간의 사건들로 정상 추적된다. 이 규칙이 없으면 "한 번 도달한 TRL은 영원하다"는 가정이 현재 상태를 체계적으로 과대평가한다.

**초기 시드(콜드 스타트 해결):** 1957년 이후 주요 마일스톤 수백 건을 히스토리컬 이벤트로 백필하고, 각 역량의 현재 TRL·휴면 여부를 초기 감사(LLM 배치 초안 + 인간 검수)로 설정한다. 첫날부터 (a) 회귀 가능한 시계열, (b) 채워진 타임라인, (c) 보정된 현재 상태를 확보한다.

## 5. 기술 필라 (6대)

| # | 필라 | 핵심 역량 예시 | 비고 |
|---|------|----------------|------|
| 1 | **수송 및 추진** | 완전 재사용 발사체, 궤도 급유, 심우주 추진(NTP/이온), 행성 착륙·이륙, 궤도 물류 | Layer B $/kg·발사 케이던스와 앵커 대조 (3.4절) |
| 2 | **생명 유지 및 인간 건강** | 폐쇄 생태계(ECLSS), 우주 농업, 방사선 방호, 미세중력·저중력 의학, 장기 체류 심리 | 테라포밍 제외 |
| 3 | **거주지 및 인프라** | 표면 기지 건설(3D 프린팅), 전력 계통, 행성 간·표면 통신망, 표면 이동 수단 | |
| 4 | **자원 및 에너지** | ISRU(추진제·물·산소), 소형 원자로/핵융합, 극한환경 소재 | Layer A(K-지수)와 장기 정합 관측 (3.4절) |
| 5 | **자율 시스템** | 무인 선행 건설 로봇, 자율 운영(항법·정비), 원격 정밀 작업 | **우주 응용에 한정.** 일반 AGI 뉴스는 게이트에서 차단 |
| 6 | **경제 및 거버넌스** | 발사 비용 하락, 민간 시장 형성, 국제 규범·거주 법제, 지속 자금 조달 | Layer B 지표 다수가 이 필라의 실측치. **TRL 대신 EGL 척도로 측정 (5.1절)** |

**통합 성숙도(SRL 개념 반영):** 다행성 정착은 시스템-오브-시스템즈이므로 개별 기술 TRL만으로는 통합 성숙도를 담지 못한다 (NASA/DoD 계열의 IRL·SRL 프레임워크가 다루는 문제). 완전한 SRL 수식을 도입하는 대신, **각 필라에 "통합 실증" 역량 노드를 명시적으로 포함**한다 — 예: "폐쇄 생태계 1년 이상 유인 연속 운전", "무인 ISRU-발사체 연계 실증". 통합 노드는 요소 기술이 모두 성숙해도 별도로 입증되어야 하므로, 필라 준비도가 요소 기술만으로 과대평가되는 것을 막는다. **통합 실증의 최종 정점은 finish line과 등치인 "26개월 무보급 연속 운영"(1절 v2.11)이다** — 이 사건 없이는 관련 통합 노드가 TRL 8+에 도달할 수 없도록 노드를 정의한다.

- 6개 필라는 UI의 육각형 레이더 차트와 일치한다.
- 역량 노드 목록은 루브릭의 일부로 버전 관리한다. 기사는 필라가 아니라 **역량 노드**에 매핑한다.

### 5.1 필라 6 전용 성숙도 척도 — EGL (Economic-Governance Level, v2.3 신설)

**문제:** TRL은 기술 성숙도 척도라 "국제 거주 법제"나 "민간 시장 형성"에는 개념적으로 적용되지 않는다.

**해법의 선례:** TRL이 재지 못하는 영역을 위해 **TRL 동형(1~N 단계) 척도를 파생시키는 것**은 이미 확립된 방법론이다 — ARENA(호주 재생에너지청)의 상업화 준비 지수 **CRI 1~6**, 미국 에너지부(DOE)의 도입 준비 수준 **ARL**, 덴마크 혁신기금의 사회적 준비 수준 **SRL 1~9**, 스티브 블랭크의 투자 준비 수준 **IRL 1~9**. 모두 "TRL만으로는 상업화·제도·수용성을 측정할 수 없다"는 동일한 문제의식에서 나온 자매 척도들이다. 본 프로젝트는 같은 패턴으로 **EGL 1~9**를 정의한다.

**설계 원칙 — 동형성(isomorphism):** EGL을 TRL과 같은 1~9 구조로 만들면, 시스템의 나머지 전부(사건 파이프라인, 검증 수준, 결정론적 채점, 준비도 사상, logit 회귀, 의존성 DAG)가 **조회 테이블 하나 교체 외에는 수정 없이** 그대로 작동한다.

| EGL | 경제 트랙 (CRI/ARL 계열) | 거버넌스 트랙 (제도 수명주기) |
|-----|--------------------------|-------------------------------|
| 1 | 사업 개념·연구 제안 | 학계·민간의 의제 제기 |
| 2 | 시장·비용 타당성 분석, 초기 투자 | 국제기구·정부의 공식 검토 의제화 (COPUOS 등) |
| 3 | 정부 주도 시범 사업·앵커 계약 | 조약·법안 초안 성립 |
| 4 | 최초 상업 서비스 매출 발생 | 채택·서명 (국제 합의 또는 주요국 입법) |
| 5 | 단일 사업자의 반복 매출·지속 운영 | 비준·발효 (법적 구속력 발생) |
| 6 | 복수 사업자 경쟁, 단가 하락 개시 | 국내 이행 입법·전담 기구 설립 |
| 7 | 정부 수요 없이 민간 수요로 자생 | 운영 관행 축적 (허가·등록 실적) |
| 8 | 비용 곡선 안정, 금융권의 표준 자산화(bankable) | 분쟁 해결 선례 확립 |
| 9 | 성숙 산업 (보험·표준·인프라 완비) | 보편적 준수, 안정적 국제 규범 |

**정량 앵커링:** 경제 트랙의 레벨 판정에는 Layer B 실측 임계값을 트리거로 결합한다 (예: EGL 6 인정 조건에 "복수 사업자 운영 + 단가 하락 추세 확인" 같은 실측 조건을 노드 정의에 명시). 판정이 뉴스 해석이 아니라 숫자에 앵커되므로, **필라 6은 역설적으로 가장 객관적으로 측정 가능한 필라**가 된다. 거버넌스 트랙의 검증도 유리하다 — 조약 발효·입법은 관보/공식 발표(Tier 1 출처)로 확정되는 사건이라 `공식 확인` 판정이 가장 쉽다.

**후퇴 규칙 (기술 필라와의 결정적 차이):** 기술 지식의 소실은 휴면(dormancy)으로 모델링했지만, 제도와 시장은 명시적으로 **후퇴**한다 — 조약 탈퇴, 법률 폐지, 기구 해산, 시장 붕괴(전 사업자 철수). 필라 6 노드는 `제도 후퇴(rollback)` 사건으로 EGL 하향을 허용한다 (검증 수준 `공식 확인` 이상 필요).

## 6. AI 평가 파이프라인

```
[수집] → [1차 게이트] → [심층 분류] → [플루크 필터(고임팩트만)] → [결정론적 채점·상태 갱신]
```

**Stage 0 — 수집:** RSS 피드 시간당 수집 + Layer B 구조화 소스(API) 별도 채널. 본문 추출, URL/해시 기반 완전 중복 제거.

**Stage 1 — 관련성 게이트 (경량 모델):** "우주 진출 역량과 직접 관련 있는가" 이진 판정. 우주 응용이 명시되지 않은 일반 기술 뉴스는 탈락(예상 탈락률 85~95%).

**Stage 2 — 심층 분류 (프론티어 모델, structured outputs):** 다음을 전부 enum 또는 원문 인용으로 추출. 자유 서술 숫자 금지.

| 필드 | 값 |
|------|-----|
| 역량 노드 | 기술 트리 노드 ID (복수 가능) |
| 사건 유형 | `이론/논문` · `실험실 성과` · `프로토타입 시연` · `비행 시험` · `실운용 배치` · `상용화` · `제도·시장 전진(institutional advance, 필라 6 한정)` · `차질(setback)` · `프로그램 취소` · `계획 발표` · `회고 보도` · `제도 후퇴(rollback, 필라 6 한정)` |
| 도달 주장 성숙도 | 기술 필라: TRL 1~9 / 필라 6: EGL 1~9 (5.1절) |
| 발표 경로 | `주체 직접 발표(1차)` · `제3자 취재 보도(2차)` · `보도자료 재전재` — **검증 수준 자체는 LLM이 판정하지 않는다** (코드 도출, 아래 규칙) |
| 주체 / 발생일 | 조직명 / 사건 실제 발생일 (보도일 아님) |
| 증거 인용문 | 판단 근거가 되는 기사 원문 발췌 (필수 — 없으면 기각) |
| 중복 판정 | 최근 사건 레지스트리 대조 결과 |

- 프롬프트: 고정 루브릭 + 역사적 앵커 예시 10~15건 (프롬프트 캐싱 대상).
- 중복 판정 2중화: (a) LLM 레지스트리 대조, (b) 코드 측 자연 키 `(역량, 유형, 주체, 발생일±7일)` + 임베딩 유사도.

**출처 신뢰 등급제와 검증 수준의 코드 도출 (v2.4 — LLM 판단에서 제거):** 소스 레지스트리에 등급을 부여하고, **검증 수준은 LLM이 아니라 코드가 결정론적으로 도출한다.** LLM은 "누가 어떤 경로로 무엇을 주장했나"만 추출하며, 그 주장이 상태를 움직일 자격이 있는지는 아래 규칙이 결정한다. 이로써 **상태 전이의 관문 전체가 비-AI**가 된다.

| 등급 | 예 | 규칙 |
|------|-----|------|
| Tier 1 | NASA/ESA/JAXA 등 기관 발표, 동료 심사 저널 | 기관 발표는 단독으로 `공식 확인`, 저널 게재는 `동료 심사` 성립 |
| Tier 2 | 전문 우주 매체 (SpaceNews, NASASpaceflight 등) | 단독 시 `주장 단계`, 독립 교차 시 승격 |
| Tier 3 | 일반 테크 미디어, 기업 보도자료 | 항상 잠정. 상위 등급 교차 확인 전까지 상태 갱신 불가 |

검증 수준 도출 규칙 (코드, 사건 클러스터 단위로 평가):

```
공식 확인 = 사건 클러스터에 Tier 1 '기관'(NASA/ESA 등)의 직접 발표가 존재
동료 심사 = 동료 심사 저널·프로시딩 게재 (프리프린트 제외 — arXiv 단독은 주장 단계)
독립 검증 = 서로 독립인 Tier 1~2 출처 2곳 이상이 각자 확인
            (보도자료 재전재는 '발표 경로' 필드로 식별해 1건으로 계산)
주장 단계 = 그 외 전부 (기업 직접 발표, Tier 3 단독, 프리프린트 단독 등)

서열: 주장 단계 < 동료 심사 < 공식 확인 < 독립 검증
복수 조건 충족 시 최고 등급을 적용한다 (v2.8 final — 기관 발표와 저널 게재를
구분하지 않으면 '동료 심사' 등급이 도달 불가능한 죽은 등급이 되는 문제 교정)
```

**Stage 2.5 — 플루크 필터 (고임팩트 사건만, v2.4 재명명):** `impact_score ≥ 8` 또는 성숙도 2단계 이상 점프 주장 사건은 별도 컨텍스트의 2차 평가를 거친다. (순서 명시 — v2.8 final: `impact_score`는 Stage 2 출력의 순수 함수이므로 분류 직후 코드가 **잠정 산출**해 이 필터의 트리거로 쓰고, 확정 채점과 상태 갱신은 Stage 3에서 수행한다.) 일치하면 인간 검수 큐로, 불일치하면 사유와 함께 우선 검수로 올린다. **이것은 "독립 검증"이 아니다** — 두 LLM 판정자는 학습 분포를 공유하는 상관된 판정자이므로, 이 단계가 잡는 것은 샘플링 요행(플루크)뿐이다. 계통 편향(예: 보도자료의 낙관적 주장을 일관되게 수용)의 방어는 이 단계가 아니라 **골든셋(인간 라벨 앵커) · Layer B(비-AI 실측 앵커) · 인간 검수 큐 · 서킷 브레이커(9절)**가 담당한다.

**Stage 3 — 결정론적 채점·상태 갱신 (코드):** 7절 규칙. LLM 불개입.

**규모·비용 (대략):** 피드 50개 × 일 40건 ≈ 2,000건/일 → 게이트 통과 5~10% → 심층 분류 100~200건/일. 게이트는 경량 모델로 일 수 달러, 심층 분류는 캐싱 적용 시 일 수~십수 달러. 플루크 필터는 일 0~수 건. 백필·재평가는 Batch API(50% 할인). 개인 프로젝트 예산으로 지속 가능.

## 7. 점수 산정 규칙 (결정론적)

표시용 0~10점은 **코드가** 계산한다:

```
impact_score = base(도달 TRL) × verify_weight × novelty
```

- `base(도달 TRL)`: TRL 1~3 → 1~3점, TRL 4~6 → 4~6점, TRL 7 → 8점, TRL 8 → 9점, TRL 9 → 10점
- `verify_weight`: 주장 단계 0.3 · 동료 심사 0.7 · 공식 확인 0.9 · 독립 검증 1.0
- `novelty`: 해당 역량의 TRL이 **실제로 전진했을 때만** 1.0, 반복 시연·후속 보도는 0. 단, **휴면 역량의 재시연은 novelty 1.0** (상태 복원도 실질 진보다)
- `계획 발표`: 상태 변화 0. 선행 지표로 별도 기록
- `차질/취소`: TRL 유지, 신뢰도·추세 하향. `실운용 배치` 프로그램의 취소는 배치 상태 해제 + 휴면 카운트다운 개시
- **필라 6(EGL) 특례:** `제도 후퇴(rollback)` 사건은 검증 수준 `공식 확인` 이상일 때 EGL을 실제로 하향시킨다 (기술 필라의 휴면과 달리 직접 후퇴). `base()` 조회 테이블은 EGL에도 동일 구조로 적용한다
- **상태 갱신 조건:** `동료 심사` 이상에서 확정 전진 (v2.8 final 교정 — 종전 "공식 확인 이상"은 논문으로만 입증되는 TRL 1~3 구간의 전진을 원천 차단하는 규칙 버그였다. 단, **도달 판정(1절)과 rollback(5.1절)은 여전히 `공식 확인` 이상**을 요구한다). `주장 단계`는 잠정 기록, 90일 내 검증 승격 없으면 소멸
- **아카이브 기준:** 역량 상태를 변화시켰거나 잠정 기록된 모든 사건
- **휴먼 인 더 루프:** 고임팩트 사건은 Stage 2.5 플루크 필터 후 인간 검수 큐 경유 (빈도 낮음). **TRL/EGL 8~9로의 전진은 예외 없이 인간 검수를 통과해야 한다 (v2.9 명문화)** — 채점 구조상 이미 보장되지만(`base(8)×0.9 = 8.1 ≥ 8`, `base(9)×0.9 = 9 ≥ 8`), 파라미터 개정으로 우회되지 않도록 독립 규칙으로 명시한다

## 8. 수학 모델

### 8.1 준비도 (Readiness)

- 역량 준비도: `r_c = trl_map(TRL_c) × dormancy_factor` — TRL 1~9를 0~1로 사상하는 후반 가중 조회 테이블 (예: TRL 1→0.03, 3→0.12, 5→0.30, 6→0.45, 7→0.65, 8→0.85, 9→1.0) × 휴면 시 기간 의존 감쇠(0.85~0.40, 4절). **필라 6 노드는 `trl_map` 대신 EGL용 `maturity_map`(동일 구조의 별도 조회 테이블)을 사용하며, 이후의 모든 수식(유효 준비도·logit·ETA)은 무수정으로 동일 적용된다 (5.1절 동형성 원칙)**
- **유효 준비도 (의존성 반영, v2.2 신설):**

  ```
  r_c^eff = min( r_c , D_c + δ_e )
  D_c  = 선행 노드들의 유효 준비도 (AND 그룹은 min, OR 그룹은 max로 집계)
  δ_e  = 간선별 허용 선행 마진 (기본 0.15 — 기술 트리 레지스트리에서 간선 단위 설정, v2.5)
  ```

  후행 노드는 선행 역량보다 최대 δ_e만큼만 앞서갈 수 있다. 선행이 없는 노드는 `r_c^eff = r_c`. DAG이므로 위상 정렬 순서로 1회 순회하면 전체 계산이 끝난다. 선행이 낮은데 후행 뉴스만 쏟아지는 하이프 상황에서 필라 준비도가 과대평가되는 것을 구조적으로 막는다.

  **δ의 간선별 차등 (v2.5):** 모든 의존성의 탄력성이 같지 않다 — 시뮬레이션 검증이 유효한 소프트웨어 계열 후행 노드(자율 시스템 등)는 선행 하드웨어 없이도 더 앞서갈 수 있으므로 해당 간선에 더 큰 δ_e를 부여할 수 있다. 단, TRL 정의 자체가 시뮬레이션 단독 진행을 TRL 5~6(관련 환경 실증 이전)에서 멈추게 하는 자연 상한으로 작동한다는 점에 유의 (예: 선행 0.30(TRL 5) + 기본 δ 0.15 = 0.45는 정확히 TRL 6 준비도와 일치). δ_e 값의 보정은 ML이 아니라 **백테스트(hindcast)**로 수행한다 — 노드 수십 개 규모의 데이터는 ML의 대상이 아니다 (13.4절).
- 필라 준비도: `R_p = Σ w_c · r_c^eff` (역량 중요도 가중 평균)
- 전체 준비도: **`R = min_p R_p` (병목 원칙)** — 대시보드에 가중 평균 병기, ETA는 병목이 지배. 유효 준비도를 쓰므로 특정 노드의 병목이 의존성 사슬을 따라 자동 전파된다

### 8.2 주 ETA — logit-선형 회귀 (로지스틱 성장 가정)

| 후보 | 판정 | 이유 |
|------|------|------|
| 선형 회귀 | ✕ | 복리적 가속 미반영 |
| 지수 회귀 | ✕ | 무한 성장 가정 → ETA 붕괴, 포화 없음 |
| **로지스틱(S-곡선)** | **◯** | 기술 성숙의 실제 형태(느림→가속→포화), TRL 유계성과 정합 |

절차 (필라별): ① 주간 스냅샷으로 `R_p(t)` 시계열 축적(백필 포함) → ② **ε-클리핑 후** logit 변환 → ③ **필라별 트레일링 윈도우 W_p(지수 감쇠 가중)의 선형 회귀**로 추세(=속도) 추정 (강건 옵션: Theil-Sen) → ④ 추세(수축·개입 처리 후의 `trend_used`, 하단 블록) 직선이 `logit(R_target)`에 도달하는 시점 역산 = ETA_p → ⑤ **전체 ETA = max_p ETA_p** (병목).

**계단형 데이터와 가짜 모멘텀 — Holt 강등 (v2.6):** TRL/EGL 준비도는 수년 평탄하다가 사건 순간에 점프하는 **계단형 시계열**인데, Holt 이중 지수 평활은 연속적 추세를 가정하므로 점프 직후 추세를 과대 추정했다가 정체기에 0으로 되끌리는 스파이크-감쇠 아티팩트(가짜 모멘텀)를 만든다 — v2.5의 사건 수 기반 수축은 이 오차의 *크기*를 줄일 뿐 *시간 구조*는 못 고친다. 근본 처방은 추정기 교체다: 트레일링 윈도우가 사건 간 간격 여러 개를 포함하도록 잡은 선형 회귀는 개별 점프의 타이밍에 둔감하고 장기 기울기만 남긴다. Holt는 ETA 경로에서 제거하고 **UI 모멘텀 지표 전용의 단기 추정기**로만 사용한다 (표시 전용 — 뉴스 직후 시계가 살짝 움직이는 연출은 감쇠 표시 층이 담당).

**필라별 윈도우 W_p (v2.7):** 윈도우의 목적이 "사건 간 간격 여러 개를 포함"하는 것이므로, 크기는 필라의 사건 빈도에 연동해 자동 설정한다:

```
W_p = clamp( m × 필라별 상태 변화 사건의 중위 간격, 4년, 15년 )    (m 기본 6)
```

소프트웨어 계열(자율 시스템)처럼 사건이 잦은 필라는 짧게, 하드웨어·인프라 계열처럼 드문 필라는 길게 잡힌다. **W_p를 백테스트로 자유 최적화하지 않는다** — 전례 없는 소표본 역사에 적중률을 극대화하도록 윈도우를 튜닝하면 그 자체가 과적합이다. 백테스트의 역할은 이 규칙의 *검증*(성능 열화 여부 확인)이지 *탐색*이 아니며, 보정 대상은 계수 m 하나로 제한한다.

**② ε-클리핑 (v2.2 명시):** `logit(R) = ln(R/(1−R))`은 `R = 0`에서 −∞, `R = 1`에서 +∞로 발산한다. 백필 초기 구간(예: 1957년의 ISRU 필라는 전 노드 미착수 → `R_p = 0`)과 finish line 근접 구간에서 실제로 발생 가능한 케이스이므로, 변환 전 반드시 클리핑한다:

```
R̃_p = clip(R_p, ε, 1−ε),  ε = 0.01 (기본값)
logit(R̃_p) = ln( R̃_p / (1 − R̃_p) )
```

- 목표값도 동일하게 `R_target ≤ 1−ε`을 보장한다 (finish line 정의상 `R_target ≈ 0.85+α`로 1 미만이지만, 클리핑이 이를 이중으로 방어).
- 로지스틱 곡선의 직선화 성질은 클리핑 후에도 유지되며, 데이터가 적은 초기에도 3-파라미터 비선형 적합보다 안정적이다. (대안: empirical logit `ln((R+ε)/(1−R+ε))` — 어느 쪽이든 발산 방지가 수식 설계에 명시되어야 한다.)

**희소 이벤트 구간의 추세 수축 (v2.5):** 클리핑은 발산만 막을 뿐, 경계 근처의 민감도는 그대로다 — `d logit/dR = 1/(R(1−R))`이므로 R이 0에 가까울수록 작은 상태 변화가 큰 logit 이동을 만들고, ETA는 추세로 나누어 역산되므로 추세가 0에 가까운 구간에서는 미세한 추세 변화가 수십 년의 ETA 요동으로 증폭된다(민감도 ∝ 1/추세²). 이 결합이 위험한 곳은 "데이터가 적은 곳"이 아니라 **"상태 변화 사건이 드문 곳"**이다 (주간 스냅샷은 개수만 많고 정보가 없다). 대응 — 사건 수 기반 베이지안 수축:

```
trend_used = (n·trend_fit + k·trend_prior) / (n + k)
trend_fit   = 트레일링 윈도우 가중 선형 회귀의 기울기 추정치 (v2.6 — 구 trend_Holt)
n           = 최근 W년 윈도우 내 해당 필라의 상태 변화 사건 수
trend_prior = 전체 필라의 역사적 평균 logit 기울기 (백필 데이터에서 추정)
k           = 수축 강도 (기본 4 — 백테스트로 보정)
```

사건이 풍부한 필라는 회귀 추세(`trend_fit`)가 지배하고, 희소한 필라는 "역사적 평균 기술 발전 속도"라는 사전값으로 수렴한다. 단일 사건에 의한 과적합 발산이 구조적으로 억제된다.

**후퇴 이벤트의 개입 분석 (v2.5):** 검증된 `rollback`·대형 `차질`은 시계열상 **수준 이동(level shift)**으로 처리하고 추세 항에는 전이시키지 않는다 — 추세 추정기가 일회성 낙하를 음의 기울기로 흡수해 "지속 하락"을 외삽하는 것을 막기 위해서다 (구현: 윈도우 회귀에 개입 시점의 수준 이동 더미 포함). 제도 붕괴는 수직 낙하하는 비대칭 사건이므로, 낙하 자체는 수준에 즉시 반영하되(정직한 표시) 추세는 체제 내(within-regime) 데이터로 추정하고 불확실성 구간을 일시 확대한다.

**구조 단절(Trend Break) — 중대 후퇴의 2단계 처리 (v2.8):** 수준 이동 처리로 충분한 것은 경미한 후퇴까지다. 성숙도 2단계 이상의 하락이나 다수 노드를 동시에 치는 중대 후퇴(예: 주요 우주 조약 탈퇴)는 단절 이전 추세의 패러다임 자체를 무효화하므로, **구조 단절을 선언하고 해당 필라의 회귀 윈도우를 단절 시점에서 리셋**한다. 신규 메커니즘은 필요 없다 — 윈도우 리셋으로 사건 수 n이 0 근처로 떨어지면 기존 수축 공식(`trend_used = (n·trend_fit + k·trend_prior)/(n+k)`)이 자동으로 `trend_prior`(역사적 평균 발전 속도)를 지배시켜, 단절 이전의 낙관적 추세 관성이 끊긴다. 불확실성 구간은 단절 후 데이터가 재축적될 때까지 확대 상태를 유지한다. **하방 충격에 인위적 하한(floor)을 두는 방식은 채택하지 않는다** — 실제 악화를 은폐해 "발산을 숨기지 않는다"는 원칙(3.4절)과 충돌한다. 표시 층의 감쇠(±90일/일)가 UX 완충을 담당한다.

가속도(추세의 변화율)는 노이즈 증폭이 심해 **ETA 계산에서 제외**, UI 모멘텀 지표로만 표시.

### 8.3 ETA 다중 대조 (앙상블 — v2.1 "삼각 측량"에서 확장)

단일 모델의 점 추정 대신 **독립적인 추정·참조치 4종(①·②·③a·③b)을 병렬 산출·병기**한다:

| 추정원 | 방법 | 성격 |
|--------|------|------|
| ① 기술 트리 ETA | 8.2절 (Layer C 기반) | 본 시스템의 주 추정치 (문명 전체) |
| ② 수송 경제성 ETA (v2.4 스코프 축소) | Layer B의 $/kg **공표 가격 곡선**을 Wright's law(누적 발사량 대비 경험 곡선)로 외삽해 필라 1의 목표 경제성(선언된 가정, 3.2절) 도달 시점 추정 | 뉴스 해석 무개입 — 단, **필라 1 한정** 교차 검증치 (문명 전체 ETA 아님) |
| ③a 군중 예측 (라이브) | Metaculus 등 예측 플랫폼의 관련 질문(유인 화성 착륙 시점 등) 커뮤니티 중앙값 | 빠르게 갱신되나 주요 인물의 발표 직후 출렁이는 하이프 민감성 — **수개월 이동 평균으로 평활해 참조 (v2.7)** |
| ③b 기관 예측 (저빈도 기준선, v2.7) | National Academies 십년 조사, NASA/ESA 공식 로드맵의 목표 연도, RAND류 연구 보고서 | 수년 주기로만 갱신되는 보수적 기준선 — 라이브 벤치마크가 아니라 느린 참조선 (목표 연도 추출 방식으로 사용) |

- 대조 방식 (v2.4/v2.7): ①의 **수송 필라 ETA ↔ ②**, ①의 **전체 ETA ↔ ③a(평활 적용)·③b**를 각각 비교하고, 각 대조의 발산 폭을 해당 불확실성 구간에 반영한다 (수렴하면 좁게, 벌어지면 넓게).
- 대시보드에 "우리 모델 vs 수송 경제성 vs 군중·기관 예측"을 나란히 표시 — 예측을 숨기지 않고 대조시키는 것이 신뢰도를 만든다.

### 8.4 불확실성 정량화 — 몬테카를로 (v2.1 강화)

단순 회귀 잔차 밴드 대신, **파라미터 섭동 몬테카를로**로 ETA 분포를 산출한다: 회귀 계수(잔차 공분산), 역량 가중치 `w_c`, `trl_map` 매핑, **그리고 백테스트 보정 파라미터(δ_e, k, 휴면 감쇠 곡선) 자체**를 수천 회 샘플링 → ETA 분포의 중앙값 + 80% 구간을 표시.

**보정 파라미터도 섭동 대상이다 (v2.7):** 다행성 개척은 전례 없는 소표본(Small N) 사건이라, 과거 궤적(정부 주도기)에 보정한 파라미터가 다른 패러다임(민간 주도 재사용 발사체 시대)의 미래에 과적합될 위험이 있다. 따라서 보정 파라미터를 점 추정으로 고정하지 않고 보정값 주변의 분포로 취급해 샘플링한다 — 파라미터 불확실성을 숨기지 않고 ETA 구간의 폭으로 정직하게 노출하는 것이다. 표시 형식: **"2047년 (2041~2062)"** — 점 추정 단독 표시 금지.

**인식론적 지위 (v2.10 명시):** 이 구간은 세계에 대한 확률이 아니라 **모델 내부의 확률**이다 — 몬테카를로가 담는 것은 파라미터 불확실성뿐이며, 모형족 선택(로지스틱)이 틀렸을 가능성이나 구조 단절(전쟁, 기술 패러다임 전환)의 미지 확률 같은 **모형 불확실성은 원리적으로 정량화되지 않는다.** 다행성 도달은 일회성·비정상(non-stationary)·행위자 구동 사건이라 빈도주의적 보정이 불가능하고, 수십 년 외삽의 사후 검증도 불가능하다(도달 전엔 못 재고, 도달하면 표본 1). 따라서 **ETA는 예보(forecast)가 아니라 "명시된 가정 하의 시나리오 투영(projection)"이다** — IPCC가 산출물을 '예측'이 아니라 '시나리오'라 부르는 것과 같은 이유. 이것이 백테스트를 궤적 재현으로 한정하고(9절) 검증 가능한 신뢰 적립을 마이크로 예측으로 분리한 설계의 근거이며, 표시 라벨은 다음으로 고정한다: **"현 추세 지속 시나리오 기준 · 모델 내 80% 구간"**.

**신뢰성 장치(유지):** ETA 클램프 [현재+2년, +150년], 표시 ETA 일일 최대 ±90일 감쇠(대형 뉴스의 가시적 이동 연출은 유지, 단일 기사의 수십 년 요동은 방지).

## 9. 신뢰도 프로그램 (Credibility Program, v2.1 신설)

"20년 뒤 예측"은 직접 검증할 수 없다. 따라서 **검증 가능한 것들을 계속 맞히는 실적**으로 신뢰를 적립한다.

1. **백테스트(hindcast):** 데이터를 과거 시점(예: 2010년)에서 절단하고 모델을 돌려 2010~현재의 준비도 궤적·중간 이정표를 재예측 → 실제와 대조해 방법론 자체를 검증. 루브릭·수학 모델 개정 시마다 재실행. **체제 경계 홀드아웃 (v2.7):** 정부 주도기(~2010)에 보정하고 민간 주도기(2010~)로 검증하는 교차 구성을 반드시 포함해, 파라미터의 체제(regime) 과적합을 탐지한다.
2. **단기 마이크로 예측 + Brier 트랙 레코드:** 시스템이 추세로부터 6~24개월 내 검증 가능한 예측을 자동 생성·공표한다 (예: "18개월 내 궤도 급유 실증 확률 70%"). **확률 생성 규칙(v2.10):** 확률의 산출 메커니즘은 사건 빈도 기반 위험률(hazard rate) 모델 계열로 하되, 구체 규칙은 구현 설계(시행계획 WP4.5)에서 정의한다 — 컨셉 단계의 미명세 갭임을 기록해 둔다. 만기 도래 시 적중 여부를 기록하고 **Brier 점수 트랙 레코드를 대시보드에 공개**한다. 단기 적중 실적이 장기 ETA의 신뢰를 담보하는 유일한 실증 수단이다. **라이브 캘리브레이션 루프 (v2.8):** 만기 예측의 오차는 ① 확률 재보정 레이어(신뢰도 곡선 기반 — 시스템이 계통적으로 과신/과소신하면 출력 확률을 단조 보정)와 ② 파라미터 드리프트 진단 경보에 연동한다. 홀드아웃 백테스트의 소표본 한계(과거 일부 필라는 TRL 1~2에 머물러 훈련 분산 부족)를 라이브 신호가 보완하는 구조다. 단, **Brier 점수를 구조 파라미터(δ_e, k, W_p)의 직접 최적화 목표로 삼지 않는다** — 공개 트랙 레코드를 튜닝 대상으로 만들면 신뢰 장치로서의 독립성이 훼손되고(굿하트의 법칙), 단기 예측 성능은 수십 년 외삽 품질의 약한 대리 지표일 뿐이다.
3. **골든셋 회귀 테스트:** 정답 합의된 역사적 기사 ~50건을 루브릭·프롬프트·모델 변경 시 및 주 1회 재평가, 분류 일치율 임계 미달 시 경보.
4. **버전 스탬프 + 원문 보존:** 모든 평가에 `{루브릭, 프롬프트, 모델}` 버전 기록. 개정 시 Batch로 전체 재평가하여 시계열 단절 방지.
5. **드리프트 관제 + 서킷 브레이커 (v2.9):** 게이트 통과율, 일일 사건 수, 점수 분포에 관리도(control chart) 적용 — 분포 이동 시 경보. **서킷 브레이커:** 골든셋 일치율이 임계 미달이거나 관리도 위반이 지속되면 **상태 갱신을 자동 동결**한다 — 신규 사건은 잠정 큐에 적체되고, 인간이 원인을 해소·확인해야 재개된다. 자동 발동(수동 아님)인 이유는 주당 2~5시간 예산에서 상시 감시가 불가능하기 때문이고, 동결이 안전한 이유는 멱등 상태 모델의 갭 내성(13.1절) 덕에 동결의 비용이 "오염"이 아니라 "며칠의 반영 지연"뿐이기 때문이다.
6. **방법론 투명성 페이지:** 채점 규칙, 수식, 레이어 간 발산 현황, 마이크로 예측 실적, 데이터 소스를 전부 공개. "이 숫자가 어떻게 나왔는가"를 사용자가 추적 가능하게 한다. **정직성 표기 4건 (v2.10 필수 게시):** ① ETA는 시나리오 투영이며 구간은 "모델 내 80%"(모형 불확실성 제외)임, ② $/kg은 공표 가격 기반 추정치임(실제 비용은 관측 불가), ③ 관측 사건(측정)과 TRL 사상·가중치(구성 지수)의 구분, ④ 수송 경제성 목표 임계값은 선언된 가정임.

## 10. 엣지 케이스 처리 규정 (요약표)

| 케이스 | 처리 |
|--------|------|
| 한 기사가 여러 필라를 다룸 | 역량 노드별 사건으로 분해, 각각 독립 평가 |
| 여러 매체의 동일 뉴스 | 사건 병합(자연 키 + 임베딩 + LLM 레지스트리). 상태 갱신은 멱등이라 병합 누락에도 인플레이션 없음 |
| 회고·기획 기사 | `retrospective` → 상태 무변화, 기존 사건에 증거 첨부 |
| 과장 보도·주장만 있는 발표 | `계획 발표`/`주장 단계` → 상태 변화 0 또는 잠정, 90일 내 미검증 시 소멸. Tier 3 단독 출처는 상태 갱신 불가 |
| 실패·폭발·프로그램 취소 | TRL 유지, 신뢰도·추세 하향. 실운용 취소는 배치 해제 + 휴면 카운트다운 |
| **장기 미사용 역량 (아폴로 케이스)** | 프로그램 종료 + 15년 무재시연 → `휴면`, 기간 의존 감쇠(×0.85에서 10년당 −0.15, 하한 ×0.40). 복원은 재시연이 입증한 수준까지 |

## 13. 운영 현실성 및 단계별 로드맵 (v2.3 신설)

본 시스템은 "무인 완전 자동화"가 아니라 **"주당 2~5시간의 운영 노동이 내장된 반자동 시스템"**으로 정의한다. 이 노동을 숨기지 않고 설계에 예산화하는 것이 지속 가능성의 전제다.

### 13.1 수집 아키텍처 원칙

1. **사이트별 파서 금지:** 기사 본문 추출은 범용 추출기(readability 계열 알고리즘)만 사용한다. 사이트 UI 개편에 깨지는 CSS 셀렉터 방식의 개별 파서를 만들지 않는다. RSS/Atom 메타데이터 자체는 표준 포맷이라 안정적이며, 실제 취약점은 본문 추출·피드 폐지·페이월 전환에 있다.
2. **구조화 소스 우선:** 같은 정보를 공식 API/데이터(arXiv API, Launch Library 2, 기관 발표)로 얻을 수 있으면 뉴스 파싱보다 우선한다. API 속도 제한은 수집 주기 설계에 반영한다.
3. **데드맨 스위치:** 피드별 신선도 SLO — 중위 발행 간격의 2배 이상 침묵 시 자동 경보. 파이프라인 상태 대시보드 상시 운영.
4. **갭 내성의 명시:** 상태 갱신 모델은 멱등이고 피드 간 중복 보도가 이중화로 작동하므로, 수집 단절의 결과는 "편향된 ETA"가 아니라 "며칠 늦은 반영"이다. 따라서 수집 장애는 긴급 대응이 아닌 주간 점검 대상으로 분류한다.

### 13.2 운영 노동 예산 (정상 운영 기준)

| 주기 | 작업 | 예상 시간 |
|------|------|-----------|
| 일간 | 자동 경보 확인 (관제 시스템이 대신 감시) | ~0 |
| 주간 | 고임팩트 검수 큐 + 드리프트 리뷰 | 1~2시간 |
| 월간 | Layer B 수동 소스 갱신 (PDF 리포트 등 비API 소스) | ~1시간 |
| 분기 | 골든셋·레이어 발산 감사 | 2~4시간 |
| 비정기 | **모델 은퇴 대응** — 핀 고정한 LLM의 지원 종료 시 마이그레이션 + 골든셋 재검증 + Batch 전체 재평가 | 연 0~2회 스파이크 |

### 13.3 콜드 스타트 작업 패키지 (일회성)

단일 최대 노동 항목이므로 별도 작업 패키지로 계획한다: ① 역량 노드 ~35개 + 의존성 간선 + 가중치(AHP) 확정, ② 1957년 이후 히스토리컬 이벤트 백필(LLM 배치 초안 + 인간 검수), ③ 초기 TRL/EGL·휴면 감사, ④ 골든셋 ~50건 라벨링, ⑤ 앵커 예시 10~15건 선정. 규모: 집중 작업 수 주. 백필 시계열(회고적으로 매끈함)과 실시간 시계열(노이즈 큼)의 분산 차이는 회귀 시 구간별 가중으로 처리한다.

### 13.4 단계별 로드맵

| Phase | 범위 | 비고 |
|-------|------|------|
| **1** | Layer C 코어: 엄선 피드 10~15개(50개 아님), 노드 ~20개, 수동 가중치, 단순 ETA+구간, 최소 대시보드 | 여기까지가 "동작하는 제품" |
| **2** | 콜드 스타트 패키지(13.3) + 관제(데드맨 스위치·관리도) + 골든셋 | 신뢰 기반 구축 |
| **3** | Layer B 수집 + K-지수(연 1회 CSV 갱신 — 전체 시스템에서 가장 저렴한 구성요소) + ETA 다중 대조(8.3절) | 빠른 승리 |
| **4** | 의존성 DAG, 몬테카를로, 마이크로 예측 + Brier 트랙 레코드, 라이브 캘리브레이션 루프(9절), 투명성 페이지, 파라미터 백테스트 보정(간선별 δ_e·수축 강도 k) | 신뢰도 프로그램 완성 |
