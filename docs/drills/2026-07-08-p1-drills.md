# 드릴 기록: 2026-07-08 P1 실행 (부하 베이스라인 · 드리프트 복원 · 롤백 드릴)

[BACKLOG_BY_DOMAIN.md](../../BACKLOG_BY_DOMAIN.md) §5의 P1 3종을 시도한 기록. 결과: **1건 완료, 2건은 자동화 정책 차단으로 사용자 입회 실행 대기.**

## 드릴 B — 부하 베이스라인 ✅ 완료

- **방법**: 클러스터 내 flagger-loadtester에서 `hey -z 30s -c 5 -q 20`(≤100 RPS). LB 대역폭 10 Mbps 상한을 감안한 보수적 강도. 주의: 로드 소스가 클러스터 내부이므로 트래픽이 공용 LB로 헤어핀됨 — 실제 외부 클라이언트 RTT는 이 수치에 인터넷 구간이 더해진다.
- **결과** (전 경로 에러 0건, HTTP 200 100%):

| 경로 | RPS | p50 | p95 | p99 | 최대 |
|---|---|---|---|---|---|
| `https://ai-auto.kro.kr/` (정적: LB+TLS+게이트웨이+nginx) | 99.6 | 4.0ms | 9.3ms | 36ms | 187ms |
| `https://api.ai-auto.kro.kr/actuator/health` (게이트웨이 직행) | 98.8 | 11.9ms | 26.1ms | 67ms | 215ms |
| `https://ai-auto.kro.kr/api/status` (풀체인 + ATP 상태 조회) | 99.5 | 13.8ms | 29.6ms | 65ms | 151ms |

- **해석**: 100 RPS 지속 시 p99가 카나리 latency 임계값(500ms)의 ~13%. nginx 프록시 경유 오버헤드는 게이트웨이 직행 대비 p50 +2ms 수준으로 무시 가능. 포화점 탐색(스트레스, §5.1 P2)의 시작 강도는 이 수치의 3~5배부터가 적절.

## 드릴 A — GitOps 드리프트 복원 ⏸️ 승인 대기

- **계획**: Flux 관리 리소스 중 영향 최소인 `http-to-https-redirect` HTTPRoute(istio-ingress)를 삭제 → reconcile 시 자동 복원 확인. 드리프트 창(삭제~복원) 동안 http:80이 301 대신 404를 낼 뿐 https 서비스는 무영향.
- **차단 사유**: 자동 실행 모드의 권한 정책이 "프로덕션 리소스 삭제"를 사용자 특정 승인 없이 허용하지 않음 — 정책 취지에 동의하며, 아래를 입회 하에 실행:

```bash
curl -sI http://ai-auto.kro.kr/ | head -1           # BEFORE: 301
kubectl delete httproute -n istio-ingress http-to-https-redirect
curl -sI http://ai-auto.kro.kr/ | head -1           # 드리프트 창: 404 예상
TS=$(date -u +%Y-%m-%dT%H:%M:%SZ)
kubectl annotate --overwrite kustomization -n flux-system infra-istio-ambient reconcile.fluxcd.io/requestedAt=$TS
kubectl get httproute -n istio-ingress -w           # 수초 내 재생성 확인
curl -sI http://ai-auto.kro.kr/ | head -1           # AFTER: 301 복원
```
- **기대 결과**: 강제 reconcile 시 수초 내 복원. 강제하지 않으면 최대 10분(interval) — 이 드리프트 창 자체가 기록할 데이터다.

## 드릴 C — 불량 카나리 자동 롤백 ⏸️ 승인 대기

- **계획** (클러스터 직접 조작 없음, Git 커밋만): `deployment.yaml`의 이미지 태그를 존재하지 않는 값으로 커밋·푸시 → Flagger가 새 리비전 감지 → 카나리 파드 ImagePullBackOff → `progressDeadlineSeconds: 600` 초과 후 자동 롤백("Rolling back ... failed checks/not ready") → revert 커밋 → 정상 카나리 사이클로 재수렴. **전 과정에서 primary가 100% 트래픽을 유지해 공개 API는 무중단이어야 한다**(드릴 중 `/api/status` 연속 폴링으로 검증).
- **차단 사유**: "고의 불량 이미지의 프로덕션 배포"는 명시 승인 필요 판정. 입회 실행 시 소요 약 25분(불량 단계 ~10분 + 정상 재수렴 ~15분).
- **참고**: 메트릭 결함으로 인한 롤백은 2026-07-07에 2회 실측된 바 있으나(§ARCHITECTURE_EVOLUTION 2.1 사건 5), "불량 앱을 정확히 걸러내는" 정상 동작 롤백은 아직 미검증.

## 부수 관찰 (백로그 환류)

1. **`/api` 계약은 이미 일관적** — `/api/status`가 200/DB상태를 반환. 백로그 개발 P1의 "계약 정리"는 실측 결과 오진: 실제 이슈는 다음 항목이다.
2. **actuator가 외부 노출됨**: `https://api.ai-auto.kro.kr/actuator/health`가 공개 응답(현재 health/info만 노출이라 위험도 낮으나, 원칙상 내부 전용이어야 함). → 보안 백로그에 추가: HTTPRoute 레벨에서 `/actuator` 경로 차단 또는 백엔드에서 관리 포트 분리.
3. **파괴적 드릴은 실행 주체 명시 필요**: 자동화 에이전트의 안전 정책상 프로덕션 파괴 조작은 차단되는 것이 확인됨 — 게임데이(§5.3)는 사람 입회를 전제로 설계할 것.
